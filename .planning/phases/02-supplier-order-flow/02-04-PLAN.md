
```typescript
        .then((data) =>
          setOrders((prev) =>
            data.map((serverOrder) =>
```

How this works:
- `setOrders` receives a functional updater so it reads `prev` (current optimistic state)
- For each order returned by the server, if `updating[serverOrder.id]` is truthy (the order is mid-mutation), keep the existing optimistic version from `prev`
- If not in-flight, use the server's version as normal
- `updating` is closed over from the component scope (line 289: `const [updating, setUpdating] = useState<Record<string, boolean>>({})`) — this is valid because `load` is redefined each time the `useEffect` runs, and React's functional updater pattern ensures `prev` is always the latest state
