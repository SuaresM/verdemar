## Component Inventory

### Existing components modified by this phase

#### CityCombobox — strict mode addition

File: `src/components/shared/CityCombobox.tsx`

Current behavior: user can blur with arbitrary text and it is accepted.

New strict behavior contract:
- Add `strict?: boolean` prop (default `false` for backward compatibility).
- On blur: if `strict=true` AND the current query text does not exactly match any `CITIES[n].city`, reset the input display to the last committed valid value (or empty string if no prior valid selection) AND show the validation error message.
- On selection from dropdown: accepted — query reflects selected city, state auto-fills, dropdown closes.
- Error message on invalid blur: "Selecione uma cidade da lista" — displayed as `text-xs text-danger` below the input (existing error prop pattern).
- Visual state during invalid: input border stays `border-gray-200` — do NOT add a red border (project pattern uses error text only, not red borders on inputs).
- Dropdown: unchanged visual — `bg-white border border-gray-200 rounded-xl shadow-lg max-h-48 overflow-y-auto`.
- Each option: `px-4 py-2 min-h-[44px] text-left text-sm hover:bg-gray-50 font-bold text-gray-800` with state shown as `text-gray-400 text-xs — {state}`.

CSS class for the input (unchanged): `w-full px-4 py-3 border border-gray-200 rounded-xl text-sm focus:outline-none focus:ring-2 focus:ring-primary/30 focus:border-primary`

#### Delivery time selector — 2-step zone + day picker

Location: `src/pages/buyer/Cart.tsx` — inside the section expanded panel.

Label: `block text-xs text-gray-500 mb-1` — text "Horário preferencial de entrega" (unchanged label text).

**Step 1 — Zone select:**
CSS: `w-full px-3 py-2 text-sm border border-gray-200 rounded-xl focus:outline-none focus:ring-2 focus:ring-primary/30`
Placeholder option: `value=""` — "Selecione a janela de entrega"
Option format: `"Seg, Qua, Sex — 07:00 às 09:00"` (3-letter day abbreviations, em-dash separator, time range with "às")

**Step 2 — Day select (appears only after zone selected):**
CSS: same as Step 1 select class.
Placeholder option: `value=""` — "Selecione o dia"
Option format: full day name — "Segunda", "Terça", "Quarta", "Quinta", "Sexta", "Sábado", "Domingo"
Container: `space-y-2` wrapping both selects.

**Loading state (zones not yet fetched):**
Single disabled select: CSS adds `opacity-50`.
Option text: "Carregando..."

**No-zones fallback:**
Element: `<p className="text-xs text-danger font-bold">`
Text: "Fornecedor ainda não configurou horários de entrega"

**Checkout button disabled states:**
- When `hasNoZones === true`: button disabled (opacity-40, cursor-not-allowed — existing disabled pattern).
- When `!section.deliveryTimePreference` (zone + day not fully selected): button disabled.
- Existing disabled class: `disabled:opacity-40 disabled:cursor-not-allowed`.
- Button label when disabled due to no zones: existing `getCheckoutLabel()` returns normal label — button is just visually disabled. No label change needed.

## Interaction Contracts

### IC-01 — CityCombobox strict blur reset

Trigger: user focuses input, types partial text, then blurs WITHOUT clicking an option.

Sequence:
1. `onBlur` fires (after 150ms delay to allow `onMouseDown` on option to win).
2. Check: does `query` exactly match any `CITIES[n].city`? (case-sensitive match against the canonical city name, not a contains check.)
3. If NO match: reset `query` state to `lastValidCity` (tracked internally) — or `""` if no prior valid selection. Call `setError('Selecione uma cidade da lista')`.
4. If YES match: accept — no error.
5. Dropdown closes regardless.

State: add internal `lastValidCity: string` ref that is updated only on `handleSelect`.

### IC-02 — CityCombobox state auto-fill (Profile.tsx)

Trigger: buyer selects a city in Profile edit mode.

Sequence:
1. `onChange(city, state)` callback fires with both city and state values.
2. Caller (`Profile.tsx`) calls `setValue('address_city', city)` and `setValue('address_state', state)`.
3. State field (`address_state`) becomes read-only display after auto-fill — buyer cannot manually edit the state when using strict CityCombobox.

### IC-03 — Delivery zone + day 2-step selection

Trigger: buyer opens Cart, section is expanded.

Sequence (happy path):
1. Zones loaded async. While `supplierZones[supplierId] === undefined`, show loading select.
2. Zones available: show Step 1 select (zone picker). Checkout button is disabled.
3. Buyer picks a zone: `handleZoneChange` fires — sets `selectedZoneId[supplierId]`, clears `deliveryTimePreference` to `""`. Step 2 select appears.
4. Buyer picks a day: `handleDayChange` fires — sets `deliveryTimePreference` to `"Quarta — 07:00 às 09:00"` format. Checkout button becomes enabled.
5. Buyer changes zone (Step 1): Step 2 resets to placeholder, `deliveryTimePreference` cleared, button re-disabled.

Sequence (no-zones path):
1. Zones fetched, `zones.length === 0`. Show danger message. Checkout button disabled.

## Mobile-First Constraints

This is a PWA with mobile-first layout. All new interactive elements must meet:

- Touch target min-height: 44px — enforced by `py-3` on inputs and CTA buttons. Combobox dropdown items use `py-2 min-h-[44px]` to guarantee 44px regardless of content height.
- iOS Safari font-size: 16px enforced by `src/index.css` media query for `select` elements — do NOT use `text-sm` alone on `<select>` without the CSS override in place (it already is).
- Dropdown max-height: `max-h-48` (192px) with `overflow-y-auto` for combobox — prevents dropdown from overflowing viewport on small screens.
- No hover-only interactions — all state changes triggered by click/tap or keyboard.
